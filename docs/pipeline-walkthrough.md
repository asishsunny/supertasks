# From Figma to Code — Pipeline Reference

Figma designs in, React pages out. Two stages, five scripts, four config files.

---

## Pipeline structure

```
code/
  scripts/       → 5 scripts that do the work
  maps/          → auto-generated (screen-registry, token-maps)
  rules/         → prompt-generated (transform-rules, compose-rules)
  artifacts/
    cache/       → raw JSX from Figma
    sections/    → clean TSX after transform
```

---

## Stage 1 — Prep (once)

Teach the pipeline about your design system and screens.

1. **generate-maps.mjs** — read reference/tokens.json → `token-maps.json`
2. **Fetch & cache** — use Figma MCP (`get_design_context`) to extract each region → `artifacts/cache/*.jsx`

After prep: raw JSX cached, maps built. Pipeline is ready.

---

## Stage 2 — Build (per screen)

```
node code/scripts/transform.mjs artifacts/cache/shell-sidebar-raw.jsx --out artifacts/sections/shell-sidebar.tsx
```

1. **Transform** — swap tokens, strip noise, detect components → `artifacts/sections/*.tsx`
2. **Compose** — hand-compose layout.tsx + pages from transformed sections + generated data
3. **Verify** — run dev server, check in browser

---

## The four config files

**maps/** (auto-generated)
1. **screen-registry.json** — where things are (screen names, section nodeIds, Figma file refs). Auto-generated by `sync-registry.mjs`.
2. **token-maps.json** — Figma token → Tailwind class lookups (colors, radius, spacing). Auto-generated by `generate-maps.mjs`.

**rules/** (prompt-generated)
3. **transform-rules.json** — component detection (Badge, Avatar, IconButton), noise stripping, font maps, imports. Prompt-generated from reference/ + sample raw JSX.
4. **compose-rules.json** — how to assemble each screen. Column defs, data variables, pagination, chart configs, imports. Prompt-generated from plan.md.

---

## What transform does

1. **Token swap** — `var(--backgrounds/bg-base)` → `bg-ui-bg-base`
2. **Spacing** — `var(--spacing-8)` → `gap-2`
3. **Radius** — `var(--radius-8)` → `rounded-lg`
4. **Font** — `var(--font/size/medium)` → `text-sm`
5. **Components** — div soup → `<Badge>`, `<IconButton>`, `<ColorAvatar>`
6. **Noise strip** — remove `aria-hidden`, `pointer-events-none`, `data-node-id`, Figma artifacts
7. **Responsive** — `w-[1440px]` → `w-full`

---

## What compose does

1. **Stat cards** — become `.map()` loop over data array
2. **Tables** — become `<Table>` with header, data rows, pagination
3. **Charts** — bars get dynamic widths from real data
4. **Imports** — collected from compose-rules + transform-rules
5. **Wrapper** — full React component with data bindings + types

---

## The five scripts

1. **prep-specs.sh** — orchestrates all prep steps (render, sync, maps, screenshots)
2. **sync-registry.mjs** — reads Figma file, writes screen nodeIds to screen-registry.json
3. **generate-maps.mjs** — reads reference/tokens.json, writes token-maps.json
4. **transform.mjs** — takes raw JSX → clean TSX using token-maps + transform-rules
5. **build-page.mjs** — orchestrator: cache check → transform → compose → format → output

---

## Config generation

Two of four config files are prompt-generated (no manual JSON editing):

**compose-rules.json** — feed Claude plan.md + data.yaml + the prompt template. Tested at 100% accuracy — zero semantic diff from hand-authored version, all screens pass typecheck.

**transform-rules.json** — feed Claude reference/ data + sample raw JSX + UI library docs. ~85-95% accuracy. Review component detection rules and noise config after generation.

Prompt templates: `prompts/generate-compose-rules.md` and `prompts/generate-transform-rules.md`
